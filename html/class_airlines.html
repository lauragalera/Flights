◆ addAirline()

void Airlines::addAirline ( const vector< string > &  dataRead)

Afageix una aerolínia vàlida al conjunt d'aerolínies.

Precondició
La mida del vector dataRead és 8. Les dades estan en el següent ordre: 
           ID, nom, alies, codiIATA, codiICAO, callsing, país, activa.
El codi ID de l'aerolínia ha de ser únic i identificatiu. Pel contrari, els codis IATA i ICAO poden repetir-se dins el conjunt.
Postcondició
Donat un vector de strings amb els valors d'una aerolínia, crea l'aerolínia i l'afageix al conjunt d'aerolínies, respectant els següents criteris: -Tota aerolínia entrada és inclosa al conjunt d'aerolínies i ha de disposar d'un codi ID únic (violació precondició en cas contrari).
-Només serà possible identificar per codi IATA les aerolínies amb camp IATA diferent de i que no comparteixin el mateix IATA amb una o altres aerolínies. Si s'acceptarà però, com identificada per IATA, aquella aerolínia que sigui activa entre un conjunt d'aerolínies inactives, tenint totes el mateix IATA.
-La identificació d'una aerolínia per ICAO és el mateix esquema que per IATA. Per tal de gestionar el segon i tercer punt, sempre, després d'afegir la última aerolínia al conjunt, cal invocar la funció finishReading.

Comprova a partir del ID o del codi IATA o ICAO d'una aerolínia si aquesta existeix en el conjunt.

Precondició
ID és -1 si es desconeix. Si es disposa dels dos paràmetres, ID i IATAICAO han de fer referència a la mateix aerolínia.
Postcondició
Retorna cert si dins del conjunt hi ha una aerolínia que tingui com a ID el valor d'entrada ID. Si aquest és desconegut, s'intenta identificar per codi IATA o ICAO. Es retorna fals quan per cap dels paràmetres s'ha pogut comprovar l'existència de l'aerolínia dins el conjunt.

◆ finishReading()

void Airlines::finishReading ( )

Neteja les dades que poden generar ambigüitats per a identificar aerolínies, ja que certs camps que haurien de ser usats com identificadors poden estar repetits.

Precondició
Cridar sempre després de llegir un conjunt d'aerolínies.
Postcondició
S'han descartat com a identificadors de membres del conjunt el codi IATA o ICAO de certes aerolínies llegides -les que comparteixen mateix codi IATA (ICAO), i no tenen una activa d'entre la resta d'inactives-, ja que no fer-ho provocaria ambigüitat per a futures tasques identificatives del conjunt.

Retorna una parella de valors on el primer indica el nom del país al qual pertany l'aerolínia i el segon el nom de l'aerolínia.

Precondició
ID és -1 si es desconeix. Si es disposa dels dos paràmetres, ID i IATAICAO han de fer referència a la mateix aerolínia, però amb ID és suficient. L'aerolínia ha d'existir dins el conjunt d'aerolínies.
Postcondició
A partir d'un indentificador de la aerolínia, bé ID -primer identificador contemplat- o bé codi IATA o codi ICAO, es retorna una parella de valors; al primer hi consta el nom del país on s'ubica l'aerolínia i al segon el nom de l'aerolínia.

◆ manageIATARepeated()

void Airlines::manageIATARepeated ( Airline *  newAirline)
private

Afageix el punter de newAirline a l'estructura _mapIATAAirlines, després de contemplar un seguit de criteris per a l'acceptació o no del codi IATA com a identificador de l'aerolínia. Actualitza el set _IATARepeated.

Precondició
newAirline és un punter a una Airline llegida.
Postcondició
Donat un punter a una nova Aerolínia newAirline, aquest s'afageix a l'estructura _mapIATAAirlines si:
  • No existeix al map cap amb aquell IATA.
  • En cas de trobar que a _mapIATAAirlines hi ha una altra aerolínia amb el mateix IATA, la nova ocupa el lloc de la primera si la nova és activa a diferència de la primera que és inactiva. I si un cas el codi IATA es troba al set _IATARepeated, aquest s'elimina perquè sí hi haurà una aerolínia amb possibilitat d'identificar.
    També es modifica el set _IATARepeated afegint el codi IATA a eliminar si:
  • En cas de trobar que a _mapIATAAirlines hi ha una altra aerolínia amb el mateix IATA, les dues són inactives, nova i primera, o les dues són actives.

◆ _mapIATAAirlines

map<string, Airline *> Airlines::_mapIATAAirlines
private

Map que té com a camp clau el codi IATA de les aerolínies i a la seva respectiva descripció hi consta un punter que apunta a aquella aerolínia. Important, en aquesta estructura només hi apareixen les aerolínies que sí es poden identificar per IATA, és a dir, no hi hauran les aerolínies amb codi IATA NULL, tampoc aquelles que comparteixen codi IATA amb altres aerolínies que com ella estan actives, o dues o més aerolínies inactives sense cap activa perquè fa impossible identificar-les per IATA.
